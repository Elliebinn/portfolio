## 건강도 96점. 콘솔 에러 0. 근데 화면에 트렌드 배지가 없었어요.

대시보드 리디자인을 끝내고 체크포인트를 돌렸어요. 스크린샷도 찍히고, HTTP 200 OK, 콘솔 에러 0건. 리포트에 "건강도 96점"이라고 나왔어요. 잘 된 줄 알았죠.

근데 실제 화면을 열어보니까 KPI 카드에 트렌드 배지(+5.2%p 같은 변화량 표시)가 없었어요. [목업](/portfolio/ko/blog/2026-03-23-aed-ai-redesign-chaos/)에는 분명히 있었는데. 기간 필터는 "전체 기간" 옵션 하나만 있었고, 사이드바 접기 버튼은 topbar 왼쪽이 아니라 footer에 있었어요.

**도구가 "통과"라고 했는데, 실제로는 빠진 게 있었어요.**

---

## 왜 못 잡았을까

체크포인트 스킬이 검증하는 건 네 가지예요.

1. 코드 변경 감지
2. before/after 스크린샷 촬영
3. 콘솔 에러 수집
4. 건강도 점수 산출

전부 "**있는 것이 제대로 동작하는가**"를 보는 거예요. 페이지가 열리는지, 에러는 없는지, 시각적으로 깨지진 않았는지.

근데 "**있어야 하는데 없는 것**"은 못 찾아요. 체크포인트는 구현된 결과물만 보거든요. 목업이라는 "정답지"와 대조하는 단계가 없었어요.

시험 채점으로 비유하면, "답을 적은 문제"만 채점하고 "아예 안 푼 문제"는 빠뜨린 거예요.

---

## 빠진 것 리스트를 뽑아봤어요

대시보드 한 페이지에서만 이만큼 빠져 있었어요.

| 영역 | 목업 | 실제 구현 | 상태 |
|------|------|------|------|
| KPI 트렌드 배지 | 5개 카드에 변화량 표시 | 없음 | MISSING |
| 유니버스 필터 칩 | 리더보드 위에 필터 | 없음 | MISSING |
| 기간 필터 | 전체/7일/30일/90일 | 전체만 | MISMATCH |
| 사이드바 토글 위치 | topbar 왼쪽 | footer | MISMATCH |

건강도 96점이라는 숫자가 무색하죠. 점수는 높은데 목업 대비 완성도는 낮았어요. 이건 도구의 관점 자체를 바꿔야 하는 문제였어요.

---

## 목업 대조 검증 추가: Step 1.5

그래서 체크포인트 스킬에 **목업 대조 검증** 단계를 추가했어요. 변경 감지(Step 1)와 스크린샷 촬영(Step 2) 사이에 들어가는 단계라서 Step 1.5라고 이름 붙였어요.

하는 일은 이래요.

1. 관련 목업 HTML 파일을 찾는다
2. 목업의 각 영역(컴포넌트, 레이아웃, 인터랙션)을 나열한다
3. 실제 구현 코드에서 대응하는 부분을 찾는다
4. **없으면 MISSING, 다르면 MISMATCH로 표시**

이 표가 리포트에 포함되니까, "작업 완료"를 선언하기 전에 빠진 게 뭔지 바로 알 수 있어요.

```
/frontend-checkpoint "대시보드 리디자인 적용"
```

실행하면 기존 before/after 비교에 더해서, 목업 대조 결과가 추가로 나와요. MISSING이나 MISMATCH가 있으면 건강도 점수와 별개로 "목업 일치도"가 따로 표시돼요.

---

## 스크린샷 방식도 바꿨어요

목업 대조를 추가하면서, 스크린샷 찍는 방식의 한계도 같이 드러났어요.

이전에는 fullPage 캡처 한 장이 전부였어요. 페이지가 길면 하단 차트 영역이 안 잡히는 경우도 있었고, 사이드바 접기 같은 인터랙션 상태는 아예 촬영을 안 했어요.

바꾼 방식은 **영역별 타겟 캡처**예요. [Playwright](https://playwright.dev/) JS로 스크롤 위치를 바꿔서 해당 영역이 화면에 보이게 한 다음 viewport를 캡처해요. 소스 코드를 건드리지 않아요.

```
스크롤 → 하단 차트 영역 캡처
사이드바 접기 클릭 → 접힌 상태 캡처
Data Sources 클릭 → 펼쳐진 상태 캡처
```

한 페이지에서 5-6장의 시나리오별 스크린샷이 나와요. "fullPage 한 장으로 대충 봤다"에서 "**기능 단위로 검증했다**"로 바뀐 거죠.

---

## Before / After (도구 자체의 변화)

| | v1 (이전) | v2 (목업 대조 추가) |
|---|---|---|
| 검증 관점 | "있는 것이 동작하는가" | "있어야 하는 것이 있는가" + "있는 것이 동작하는가" |
| 목업 대조 | 없음 | MISSING / MISMATCH 자동 탐지 |
| 스크린샷 | fullPage 한 장 | 영역별 타겟 캡처 (5-6장) |
| 인터랙션 촬영 | 기본 상태만 | 클릭/스크롤 시나리오별 촬영 |
| 리포트 | before/after + 건강도 | + 목업 일치도 + 시나리오별 비교 |

건강도 96점인데 MISSING 4개 — 이런 상황을 이제는 잡아낼 수 있어요.

---

## 이런 상황에서 쓰면 좋아요

- 목업(HTML, Figma)이 있는 상태에서 **구현 완료 여부를 검증**할 때
- "다 했다고 했는데 빠진 게 있었다"를 **배포 전에 잡고 싶을 때**
- 페이지가 길어서 **fullPage 캡처로 하단 영역이 안 잡힐 때**
- 인터랙션(토글, 드롭다운, 탭)이 포함된 변경을 **기본 스크린샷만으로 검증하기 어려울 때**

---

## 현재 v2. 다음은 CI 연동

지금은 수동으로 `/frontend-checkpoint`를 실행해야 목업 대조가 돌아가요. 다음에는 CI에 붙여서 PR마다 자동 실행되게 만들 생각이에요. PR을 올리면 변경된 페이지의 목업 대조 결과가 자동으로 코멘트에 달리는 거죠. MISSING이 있으면 머지를 막을 수도 있고요.

fullPage 한 장짜리 건강도 체크에서 시작해서, 목업 대조 + 영역별 타겟 캡처까지 왔어요. 도구를 쓰면서 한계가 보이고, 그걸 고치면서 다음 버전이 나오는 흐름. 다음 버전은 사람이 실행 버튼을 안 눌러도 되는 걸 목표로 하고 있어요.
